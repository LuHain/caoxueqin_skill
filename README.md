# 曹雪芹.Skill

这是一个用于《红楼梦》曹雪芹相关的结构化 skill，支持章回批注、文本比较、人物与主题分析，以及在严格约束下进行**八十回后续推演**。


## 这个 skill 能做什么

- 为章回内容添加文本、历史与解释性批注
- 比较不同抄本与版本，如脂评本、庚辰本、甲戌本等
- 处理有争议的解读，并在修订中保留证据意识
- 在只依赖前80回及相关材料的前提下，进行受约束的后续推演

## 如何使用这个 skill

### 1. 一般使用

当任务涉及以下内容时，可以直接调用 `caoxueqin-skill`：

- 章回解读或批注
- 异文、版本、抄本比较
- 曹雪芹生平与史料状态
- 人物分析
- 红学解释
- 学术风格的写作与改写

示例调用：

```text
用 caoxueqin-skill，帮我解读第5回这一段，并注明依据。
```

```text
用 caoxueqin-skill，比较庚辰本和甲戌本这里的异文。
```

```text
用 caoxueqin-skill，用 scholarly note 的方式写一段关于黛玉病体意象的分析。
```

### 2. 前80回约束下的后续推演

如果你要做80回后的推演、续写、提纲或人物走向预测，调用时要把约束说清楚：

- 使用 `caoxueqin-skill`
- 不读后40回
- 以前80回正文、相关批语和 `open_threads_template.md` 为基础

强默认调用句：

```text
用 caoxueqin-skill，按前80回正文、前80回脂批和 open_threads_template，禁止读取后40回，先列 Known Setup / Open Threads / Probable Continuations / Boundary Notes，再写后续大纲。
```

常用变体：

```text
用 caoxueqin-skill，不读后40回，只推演宝玉和黛玉在80回后的走向。
```

```text
用 caoxueqin-skill，不读后40回，给我一个 10 到 12 回的详细续写分回提纲。
```

```text
用 caoxueqin-skill，不读后40回，按 continuation_builder prompt 写一段 post-80 的续写草稿。
```

建议追加的限定语：

- `只做大纲，不写正文。`
- `先列 Known Setup / Open Threads / Probable Continuations / Boundary Notes。`
- `用 scholarly note 的方式写。`
- `用 style-guided sequel sketch 的方式写。`
- `只用前80回能支持的内容，不要装作知道原作结局。`

### 3. 与后40回做对比的比较模式

默认情况下，后40回不应被读取或依赖。

只有在用户明确要求时，才读取或比较81至120回，例如：

- 比较前80回约束推演与程高本后40回的差异
- 比较人物后续走向的不同处理方式
- 比较结构、风格或结局机制的差别

示例调用：

```text
用 caoxueqin-skill，对比前80回约束下的宝黛后续推演，和资料里后40回的处理有什么不同。
```

## 工作流程概览

这个 skill 的基本路径是：

`intake -> evidence -> interpretation -> style -> continuation -> guardrails -> output`

快速说明：

- `SKILL.md` 是总说明与操作契约。
- `CALL_CARD.md` 提供可直接复制的续写调用句。
- 做 post-80 推演时，优先使用 `corpus/user_notes/open_threads_template.md`。
- 结果应按类型保存到 `outputs/chapter_notes/` 或 `outputs/draft_sections/`。

## 续写模式的约束原则

在“前80回约束续写”模式下：

- 以前80回为主要叙事依据
- 以前80回相关批语和注释为主要评论依据
- 除非用户明确要求比较，否则不要把高鹗续书情节偷带进推演
- 不要把任何推演说成“曹雪芹原来一定就是这么写的”
- 要区分明确铺垫、强推断、弱推测与开放不定之处

这个模式的目标是“受约束的重建”，不是“恢复失落原稿”。

## 关于后40回的说明

这个仓库里可能已经包含 `corpus/original_texts/chapters/` 下的81至120回。
但对于 continuation 类任务，它们应被视为受限制材料，除非用户明确要求对比。

一旦进入比较模式，可以用下面的框架来理解：

- 前80回更支持结构压力、情绪递变和开放式推演
- 后40回更倾向于把这些压力转换成明确事件链与既定结局

## 示例：前80回约束下的宝黛推演，与资料中后40回的区别


### 相合处
- 我前面说“园中气氛先冷，关系在外部压力下越来越难安放”，这一点和后40回很接近。第81回里宝玉因迎春事大哭，直奔潇湘馆，已经明确感到“园中光景，已经大变了”，黛玉也立刻被触动，这和我说的“气氛先于事件本身变冷”基本一致。ch081.txt

- 我说“黛玉会更早意识到情不能久安，而且会更内收、更病重”，这和后40回也高度相合。第83回写她因窗外一句骂声便疑心到自己身上，几乎“肝肠崩裂”；第97回到第98回则把这种内伤一路推进到焚稿、吐血、临终。ch083.txt ch097.txt ch098.txt

- 我说“宝玉会在失去黛玉后发生根本性动摇”，后40回也用了这个基本逻辑。第98回里宝玉病势大作，直言要与黛玉“活着也好一处医治伏侍，死了也好一处停放”，得知黛玉死讯后几乎神魂崩解，这和“黛玉之失成为宝玉精神转折核心伤口”是相合的。ch098.txt

### 分叉处
- 最大的区别，是我前面的推演刻意保留“未成全”的开放性，不把它落实成单一、明确、戏剧化的婚变结构；而后40回则把宝黛线强行压缩成一个非常具体的事件链：议婚误导、宝玉误认新妇、宝钗成婚、黛玉同夜而死。这个处理非常“定局”，而不是“开放的结构压力”。ch097.txt ch098.txt

- 第二个区别，是我前面强调“二人的交流会越来越隐约，靠更少的相聚、更重的言外之意来完成”；后40回虽然前段还有这种味道，但到第97至98回时，已经明显转成大事件驱动和强情节收束，尤其“焚稿”“同时成婚与死亡”这些安排，象征很强，但也更像续书作者主动完成悲剧，而不是前80回那种层层积压、迟迟不决的笔法。ch082.txt ch097.txt

- 第三个区别，是我没有把宝玉后续写成会逐渐把情感转移到宝钗身上；而后40回在第98回明写宝玉“渐渐的将爱慕黛玉的心肠略移在宝钗身上”，第99回又进一步写成夫妻间已有调笑亲近。这个转折在续书内部是为了让宝玉回到家庭秩序里，但从前80回的宝黛结构看，这一步非常快，也明显更偏程高本的伦理收束，而不是我前面那种“黛玉成为宝玉后半生不可复得的原点”。ch098.txt ch099.txt

- 第四个区别，是我前面把黛玉之死的意义放在“情深而不偶”的结构完成上，没有替她设计过强的指向性怨恨；而后40回里，第98回至第99回把黛玉临终处理得更尖锐，甚至借凤姐之口说她“临死咬牙切齿倒恨着宝玉呢”。这是一种强烈、明确、便于戏剧传播的写法，但比前80回里黛玉的复杂性更单向，也更像续书的情感定性。ch098.txt ch099.txt

### 一句话总结
- 前80回约束下的推演，重心在“关系如何被世界慢慢压成未成之局”；后40回的资料，重心在“把这个未成之局迅速落实成婚配错置、黛玉之死、宝玉病倒并被重新纳回秩序”的一套明确情节。前者更开放、更含蓄；后者更定局、更戏剧化。
